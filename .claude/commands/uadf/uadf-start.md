# UADF Session Start

## Orientation

Read `SESSION-HANDOFF.md` to get the last session's state and next steps. MEMORY.md is auto-loaded — skip it.

Also check:
```bash
git status
git log --oneline -5
```

**Do NOT read `JOURNAL.md`** — it's an append-only archive. Only consult it if a task requires understanding historical decisions.

## Report

After reading, provide a brief status report:

```
## Current State

**Branch:** [current branch]
**Last Session:** [date from handoff]
**Last Completed:** [what was done]

### Next Steps (from handoff)
1. [Next action]

### Blockers
- [Any blockers or open questions]
```

## Then Ask

"What would you like to work on this session?"

Suggest options based on the current state — continuing from the handoff, addressing a blocker, or starting something new.

## Reminders

- Memory lives in files, not chat
- Use `/uadf-handoff` before ending a session
